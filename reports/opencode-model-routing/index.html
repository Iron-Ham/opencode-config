For every reported score, higher is better and rank 1 is best. Quality:Value cubically weights judged quality and then applies cost efficiency. The weighted metric is 60% cubic quality, 25% cost efficiency, and 15% speed efficiency. Invalid or noncompliant artifacts may retain descriptive ranks, but cannot win a promotion decision.

02

Production implementation

Workload shape changes the winner.

The final matched gate retains Terra provisionally under a locked decision rule while identifying Sol as the stronger measured quality challenger. This is a risk-aware policy—not a universal model ranking.

Current evidence hierarchy

Validation and production handoff come first. Quality, cost, and speed rank only the artifacts that remain viable.

Terra leads the access-control migration, legacy-client permission UI, and admin-capability gating workloads on the weighted metric; GLM leads release orchestration; Sol leads cross-service access IaC and produced the only mergeable artifacts on the iOS comment-identity and Android persisted-thread workloads.

Final matched gate Two repeats per route ran on each of three production-shaped tasks: native editor, native offline/data, and modern client. Every attempt completed; there were no provider terminal failures or benchmark wall-time limits.

Frozen packet provenance Native editor used judge-packet schema v1; native offline/data and modern client used schema v2. Historical judge outputs were not silently regenerated after the harness changed. Final source reconciliation and scoring use the retained outputs under a common decision process; future matched runs use schema v2.

Validity definition A valid result completed the candidate run, passed the declared policy boundary, and passed effective deterministic validation. Invalid artifacts retain blind scores and handoff assessments for diagnosis, but cannot qualify for promotion or the cost-per-valid denominator.

Scope boundary These single-attempt cohorts support workload-specific findings, while the completed repeated switch gate drives the provisional default decision. Representative-workload expansion remains necessary before treating the result as universal. View the historical aggregate.

Coverage boundary Cohort labels name one workload, not an entire platform or work across engineering. The current frontend task is a small legacy-client change, the original iOS route matrix is confined to native comment identity, and the Android shortlist is concentrated in AI chat. Broad routing claims across engineering need broader iOS CRDT, IME, and editor-interaction coverage beyond the completed matched editor/offline tasks, plus query execution, agent-runtime protocols, external-API security contracts, realtime distributed state, complex client/accessibility work, and Android work outside AI.

05

Study configuration

The measured local workflow.

This records the user-level configuration used for the study. It is not a repository-level OpenCode default across engineering.

  1. Controller routingBuild, plan, and Ultra use Terra xhigh. Personal Luna, Terra, Sonnet, and Sol slash commands are not installed; those evaluated models remain available through explicit model selection. The optional /advise lane currently uses Opus 4.8 xhigh.
  2. Iteration policyBuild, plan, Luna, Terra, Sonnet, Sol, and Ultra have no OpenCode step cap.
  3. Goal safety railsOrdinary Goal runs may use configured limits. Ultra explicitly creates a durable Goal without a default token, turn, or elapsed-time cap; provider-failure and no-progress loop guards still apply.
  4. Advisor consentThe legacy advisor tool is denied everywhere. The isolated /advise lane is developer-invoked, read-only, configurable, and disableable.
  5. CompactionThe active session model compacts; old tool output is pruned, three recent turns and 12,000 recent tokens are preserved, and 20,000 tokens are reserved. It is not a separately routed role.
  6. Open-weight availabilityExact Baseten and Fireworks Standard/Fast commands are selectable, but remain outside default models, small_model, and automatic Task routing.
06

Limits and next decisions

Useful now. Designed to be overturned.

  1. No full shipping loop.The implementation cohorts produced and validated patches, but did not complete an edit-build-simulator-CI-release sequence for every route.
  2. Small and uneven production sample.The seven historical implementation cohorts have one attempt per route, while the switch cohort has six attempts per route across three tasks. Several large product-risk surfaces remain unsampled; this supports a provisional routing policy, not a universal model ranking across engineering.
  3. The original source-analysis routes did not pass 85.Several implementation artifacts did, but no model substitutes for builds, tests, runtime evidence, independent review, or human ownership.
  4. Incomplete attempts have lower-bound cost.The historical source-analysis Luna xhigh and Terra max figures exclude unfinished future work and are not complete route totals. The separate iOS Terra max attempt is reported as a censored one-hour route with its retained patch and observed cost.
  5. Luna high is workload-sensitive.It led the mixed value frontier and was inexpensive in frontend, but scored poorly in tooling, failed IaC validation, and produced a redrive iOS artifact; keep it bounded.
  6. Planner-to-implementer effect unmeasured.Terra produced the most stable plans; this study did not execute each plan with a cheaper downstream implementer.
  7. Open-weight evidence is preliminary.Matched Baseten outcomes now exist for several workloads and the ten added references passed calibration, but the full cross-area candidate matrix and provider comparison are incomplete. Vendor token-throughput claims do not establish end-to-end OpenCode speed.
  8. List price is not a bill.Subscription terms, discounts, caching, and regional or long-context rules can change out-of-pocket spend.
  9. Sol effort is deferred.High versus xhigh versus max remains a separate experiment.
  10. No carry-as-is parity.None of the 12 matched attempts cleared source reconciliation without repair. The gate measures relative route performance, not proof that either route can replace production verification.
