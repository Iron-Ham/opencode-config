Pi versus OpenCode

OpenCode wins. Luna gets the keyboard.

For an overwhelmingly iOS workload, the sensible default is Luna xhigh without routine review: full measured Swift quality at the lowest cost. Sonnet remains the research and recovery lane; Sol remains an approval-gated advisor.

Choose OpenCode.
Default to Luna. Build / general: GPT-5.6 Luna xhigh
Research: Sonnet 5, provider default
Advisor: GPT-5.6 Sol xhigh, approval-gated
Bottom line

Luna, Terra, and Sonnet each passed all three hidden Swift trials at 100/100. Luna cost 69% less than Sonnet on average; Terra was fastest but cost about twice as much as Luna.

01

Recommendation

Optimize the routine path, preserve strong escape hatches.

OpenCode is the daily driver. Luna xhigh owns routine iOS implementation. The expensive models are available on purpose, not paid on every task.

Default: use GPT-5.6 Luna xhigh for build and general, without an advisor. It matched Terra and Sonnet on every public and hidden Swift behavior in three controlled runs while posting the lowest mean and median cost.

Escalation: use /terra when turnaround matters, /sonnet for broad repository forensics or after Luna stalls, and /ultra for the highest-risk work. Sol review is approval-gated because it improved two of three drafts but materially harmed one.

02

Harness and ecosystem

Pi has a catalog. OpenCode has the better daily-driver fit.

Both are open source. Pi is a deliberately small core with an unusually rich extension surface; OpenCode offers more of the agent, permission, and provider machinery this workflow needs before customization.

5,296

Packages visible in the official Pi catalog on 14 July 2026. The live count will change.

Pi

Packages bundle extensions, skills, prompt templates, and themes, with searchable type, popularity, recency, and install metadata. Goal loops, subagents, permission layers, and multi-agent orchestration are discoverable products rather than scattered examples.

OpenCode

Awesome OpenCode is a curated GitHub list, not a first-party package catalog. OpenCode can install npm plugins and discover skills, but it does not match Pi's unified registry experience.

Pi wins extension discovery; OpenCode wins this deployment.

Pi's own documentation warns that packages run with full system access. This setup benefits more from OpenCode's native allow/ask/deny permissions and a smaller curated plugin set.

What the source actually does

OpenCode 1.17.20 routes GPT-family models to its Codex-derived prompt, Claude models to an Anthropic-specific Task/Todo prompt, and unrecognized families such as GLM to a generic prompt. A custom agent body replaces the family prompt; repository instructions and skills are then added.

The GPT lane is closest to Codex's autonomy and dirty-worktree discipline, though it names a parallel tool OpenCode does not expose. The Claude lane matches OpenCode's Read/Edit/Task/Todo tools more literally but carries less of Codex's persistence and completion contract. The generic lane asks more of the model.

Pi's raw prompt is intentionally tiny—roughly tools, command preferences, and concision. That is an excellent substrate for extension authors and the least governed out-of-box environment for long, mutation-heavy work.

Model implication: proprietary models with strong native tool use can compensate for a thinner harness; open-weight models are more sensitive to prompt/tool mismatch. GLM 5.2 therefore remains a bounded opt-in worker with deterministic verification, not a silent controller default.

Predicted out-of-box reliability: Codex first; OpenCode's GPT lane next; OpenCode's Claude lane after that; OpenCode generic and raw Pi most dependent on added policy.

This is a source-based inference, not a controlled cross-harness score. A carefully composed Pi package stack can change the ordering.

Protocol and invalid-run handling
Each run used a private copy of the same Swift package, Latin-square ordering, read/glob/grep/edit only, a fixed external test harness, a network-denied macOS sandbox, isolated SwiftPM state, and a hidden test added only after the final turn. An earlier v2 run is excluded because inherited PWD caused edits to the canonical fixture. The retained runner fixes --dir, cwd, PWD, and INIT_CWD; guards absolute paths; copies from a read-only private snapshot; and verifies the fixture hash.

Advisor protocol limits
This was a staged draft → tool-less critique → controller revision protocol using the installed Advisor's static system prompt and transcript serialization. It was not a native in-turn Advisor tool call. The screen used one offline-pagination investigation; the finalist extension used two bridge-concurrency repetitions. Answers were randomized and blind graded by two or three independent graders. Route costs include the shared draft counterfactually; recorded experiment spend counts each shared draft once. Valid advisor experiment spend was $5.255; valid Swift implementation spend was $1.905.

08

Decision boundaries

Strong evidence, narrow claims.

  1. The Swift sample is small.Three runs per model on one concurrency-sensitive package are enough for a personal default, not a general intelligence ranking across all iOS work.
  2. The advisor sample is smaller and staged.Three matched drafts across two forensic tasks expose both lift and harm. The protocol mirrors the advisor prompt and transcript shape but is not a native in-turn tool call.
  3. Advisor egress needs consent.Luna- or Terra-to-Sol stays inside OpenAI; Sonnet and Ultra cross from Anthropic to OpenAI. The rules skip the gate when the transcript is not approved for that transfer.
  4. Prompt/tool mismatches still exist upstream.The GPT prompt names a nonexistent parallel tool. Non-GPT models receive Edit/Write while GPT models receive apply_patch; the user rules normalize that difference.
  5. Compaction changes the cost envelope.Long Luna sessions compact through Sonnet, so the default route is not provider-contained and its total cost can include an Anthropic request.
  6. Codex app runtimes do not all port.Browser/computer use, documents, spreadsheets, presentations, Gmail, Calendar, Workspace Agents, and GitHub connectors remain Codex-only; compatible skills and Notion MCPs do port.
  7. Pi packages are code, not prose.A future Pi experiment should choose one coherent orchestration suite, audit its source, and avoid overlapping lifecycle extensions.
  8. OpenCode is moving quickly.Prompt routing, model metadata, plugin APIs, and tool exposure are version-specific observations pinned to OpenCode 1.17.20 and the reviewed source commit.

Methodology limits
Vendor benchmark numbers were not used as direct rankings because harnesses, tools, prompts, and evaluation policies differ. The implementation test covers one bounded Swift concurrency task; the advisor test covers two read-only iOS investigations. Timed-out draft costs are lower bounds. The conclusion is deliberately personal and operational: it chooses a default for this workload and configuration rather than claiming one model is universally best.
